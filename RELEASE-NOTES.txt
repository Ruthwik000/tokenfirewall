WHAT'S NEW
================================================================================

🚀 CROSS-PROVIDER FALLBACK (Major Feature)

Automatically switch between OpenAI, Anthropic, and Gemini when your primary
provider fails. This provides maximum resilience for production applications.

Key Benefits:
  ✓ Automatic failover across different LLM providers
  ✓ Transparent request/response transformation
  ✓ Unified budget tracking across all providers
  ✓ Zero code changes required
  ✓ Production-ready with comprehensive error handling

Example:
  If GPT-4 fails → automatically retry with Claude → then Gemini
  All transparent to your application!

================================================================================
NEW APIS
================================================================================

1. registerApiKeys(keys)
   Register API keys for multiple providers to enable cross-provider fallback.
   
   Example:
   registerApiKeys({
     openai: process.env.OPENAI_API_KEY,
     anthropic: process.env.ANTHROPIC_API_KEY,
     gemini: process.env.GEMINI_API_KEY,
   });

2. isCrossProviderEnabled()
   Check if cross-provider fallback is currently enabled.
   
   Example:
   const enabled = isCrossProviderEnabled();
   console.log("Cross-provider enabled:", enabled);

3. ModelRouterOptions.enableCrossProvider
   New option to enable cross-provider fallback in the router.
   
   Example:
   createModelRouter({
     strategy: "fallback",
     fallbackMap: {
       "gpt-4o": ["claude-3-5-sonnet-20241022", "gemini-2.5-pro"]
     },
     enableCrossProvider: true,  // <-- Enable cross-provider
     maxRetries: 2
   });

QUICK START - CROSS-PROVIDER FALLBACK
================================================================================

const {
  createBudgetGuard,
  createModelRouter,
  registerApiKeys,
  patchGlobalFetch,
} = require("tokenfirewall");

// 1. Register API keys for multiple providers
registerApiKeys({
  openai: process.env.OPENAI_API_KEY,
  anthropic: process.env.ANTHROPIC_API_KEY,
  gemini: process.env.GEMINI_API_KEY,
});

// 2. Create budget guard
createBudgetGuard({
  monthlyLimit: 100,
  mode: "block"
});

// 3. Create router with cross-provider enabled
createModelRouter({
  strategy: "fallback",
  fallbackMap: {
    "gpt-4o": ["claude-3-5-sonnet-20241022", "gemini-2.5-pro"],
  },
  maxRetries: 2,
  enableCrossProvider: true,  // Enable the magic!
});

// 4. Patch global fetch
patchGlobalFetch();

// 5. Use any LLM API - fallback is automatic!
const response = await fetch("https://api.openai.com/v1/chat/completions", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${process.env.OPENAI_API_KEY}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    model: "gpt-4o",
    messages: [{ role: "user", content: "Hello!" }]
  })
});

// If OpenAI fails, automatically retries with Claude, then Gemini!

================================================================================
HOW IT WORKS
================================================================================

1. DETECT FAILURE
   Router detects when primary provider fails (rate limit, outage, etc.)

2. TRANSFORM REQUEST
   Automatically converts request to backup provider's format
   Example: OpenAI format → Anthropic format

3. SEND TO FALLBACK
   Uses appropriate API key for backup provider

4. TRANSFORM RESPONSE
   Converts response back to original provider's format

5. RETURN TRANSPARENTLY
   Your code receives response as if primary provider answered!

================================================================================
SUPPORTED TRANSFORMATIONS
================================================================================

Request Transformations:
  ✓ OpenAI → Anthropic → Gemini
  ✓ Anthropic → OpenAI → Gemini
  ✓ Gemini → OpenAI → Anthropic

Response Transformations:
  ✓ Anthropic → OpenAI (content, tokens, finish reasons)
  ✓ Gemini → OpenAI (candidates, usage metadata)
  ✓ OpenAI → Anthropic (messages, system field)

All transformations preserve:
  ✓ Message content and roles
  ✓ Token usage for accurate cost tracking
  ✓ Temperature and max_tokens settings
  ✓ Special characters and formatting

================================================================================
UNIFIED BUDGET TRACKING
================================================================================

All costs are tracked in ONE unified budget, regardless of which provider
actually handled the request:

const status = getBudgetStatus();
console.log(`Total spent: $${status.totalSpent.toFixed(4)}`);
console.log(`Across: OpenAI, Anthropic, Gemini`);

COMPATIBILITY
================================================================================

✓ Fully backward compatible with v2.0.1
✓ Cross-provider is opt-in (enableCrossProvider: false by default)
✓ Existing code works without any changes
✓ No breaking changes to existing APIs

To enable cross-provider fallback:
  1. Set enableCrossProvider: true in createModelRouter()
  2. Register API keys with registerApiKeys()
  3. That's it! Everything else is automatic.

================================================================================
LIMITATIONS
================================================================================

Current limitations of cross-provider fallback:
  - Streaming requests not supported (falls back to standard fetch)
  - Function calling not yet supported (basic chat works perfectly)
  - Vision/multimodal not yet supported (text-only transformations)

These limitations will be addressed in future releases.
